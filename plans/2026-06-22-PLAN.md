# Component 1: Clickable Billboard Sprite with Audio

## 1. Problem Statement

### Use Case

The AR Tour Guide application needs interactive points of interest that can be placed in the 3D world. These points should always remain readable by facing the user and should allow interaction through tapping or clicking.

In the final application, these billboards will act as historical points of interest, tour markers, or character representations that can play audio stories when selected.

### Problem Being Solved

Standard Three.js sprites always fully face the camera, including pitch and roll. This can look unnatural in AR because the object tilts with device movement.

We need a reusable component that:

- Always faces the user horizontally.
- Remains upright.
- Supports click/tap interaction.
- Triggers audio playback.
- Can later be reused by the AR viewing scene.

This component should be completely independent from GPS, Redux, WebXR, tours, and asset loading.

---

# 2. Goals

## Functional Requirements

The component must:

- Render a 2D image in 3D space.
- Always face the camera.
- Rotate only around the Y axis.
- Stay upright regardless of camera pitch.
- Detect click/tap events.
- Play an audio clip when selected.
- Support multiple billboard instances.

---

## Non-Functional Requirements

The component must be:

- Reusable in any Three.js project.
- Independent of AR features.
- Independent of GPS.
- Independent of Redux.
- Unit testable.
- Easy to integrate into later components.

---

# 3. Success Criteria

The component is considered complete when:

- Billboards always face the camera horizontally.
- Pitch and roll remain zero.
- Clicking a billboard triggers playback.
- Multiple billboards can coexist.
- Unit tests pass.
- Demo scene works on desktop.

# 5. Rendering Approach

## Option 1: THREE.Sprite

Advantages:

- Built-in billboard behaviour.

Disadvantages:

- Faces camera on all axes.
- Cannot easily remain upright.

Result:

Not recommended.

---

## Option 2: Textured Plane Mesh

Example:

```ts
const geometry = new THREE.PlaneGeometry(1, 1);

const material = new THREE.MeshBasicMaterial({
  map: texture,
  transparent: true,
});

const mesh = new THREE.Mesh(geometry, material);
```

Advantages:

- Full control over rotation.
- Supports cylindrical billboarding.
- Easier to test.

Result:

Recommended approach.

---

# 6. Billboard Rotation Logic

## Desired Behaviour

The billboard should:

- Face the camera.
- Rotate only around Y.
- Ignore camera pitch.
- Ignore camera roll.

---

## Example Algorithm

```ts
const dx = cameraPosition.x - spritePosition.x;

const dz = cameraPosition.z - spritePosition.z;

const rotationY = Math.atan2(dx, dz);
```

Apply:

```ts
mesh.rotation.set(0, rotationY, 0);
```

---

## Example

Camera:

```ts
10, 5, 10;
```

Billboard:

```ts
0, 0, 0;
```

Expected:

```ts
rotation.x === 0;
rotation.z === 0;
```

Only Y changes.

# 7. Audio System

## Behaviour

When user clicks:

1. Raycaster detects hit.
2. Billboard becomes selected.
3. Audio starts playing.

---

## State Flow

```text
IDLE
  ↓ click
PLAYING
  ↓ audio ends
IDLE
```

---

## Alternative Considerations

### Toggle Playback

```text
IDLE
 ↓ click
PLAYING
 ↓ click
PAUSED
```

More complex.

Not required for first version.
