# .dependency-cruiser.cjs

## Purpose

Dependency-cruiser configuration for architectural boundary enforcement.

## Rules

| Rule                      | From          | To       | Description               |
| ------------------------- | ------------- | -------- | ------------------------- |
| `no-circular`             | any           | circular | No circular dependencies  |
| `no-storage-importing-ui` | `src/storage` | `src/ui` | Storage is lower layer    |
| `no-sensors-importing-ui` | `src/sensors` | `src/ui` | Sensors is lower layer    |
| `no-state-importing-ar`   | `src/state`   | `src/ar` | State is pure, no AR deps |
| `no-state-importing-ui`   | `src/state`   | `src/ui` | State is pure, no UI deps |

## Architecture Layers

```
ui (top)
  ↓
ar, sensors
  ↓
state, storage (bottom)
```

Lower layers cannot import from higher layers.

## Usage

```bash
npm run check:boundaries
```

## Visualization

```bash
depcruise -c config/.dependency-cruiser.cjs --output-type dot src | dot -T svg > deps.svg
```
