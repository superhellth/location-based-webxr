# `.jscpd.json` — Copy/Paste Detection Config

## Purpose

Configures [jscpd](https://github.com/kucherenko/jscpd) to detect copy-paste (duplicated) code across the codebase. Duplicate code is a maintainability risk—changes must be replicated manually and bugs can hide in divergent copies.

## Key Settings

| Key         | Value                                   | Meaning                                                     |
| ----------- | --------------------------------------- | ----------------------------------------------------------- |
| `threshold` | `1`                                     | Max percentage of duplicates allowed before the check fails |
| `minTokens` | `50`                                    | Minimum token length to consider a block a duplicate        |
| `reporters` | `["console"]`                           | Output format (console-only for CI feedback)                |
| `files`     | `["src/**/*.ts"]`                       | Only scan TypeScript source files                           |
| `ignore`    | `[...tests, .d.ts, node_modules, etc.]` | Exclude tests, declarations, and generated output           |
| `gitignore` | `true`                                  | Respect `.gitignore` patterns                               |

## Invariants & Assumptions

- **CI blocks on threshold breach**: If >1% duplication is detected, the check fails.
- **Tests are excluded**: Duplication in tests is more acceptable (test-per-scenario isolation).
- **Type declarations excluded**: `.d.ts` files often repeat signatures legitimately.

## Usage

```bash
# Via npm script
npm run check:dup

# Direct invocation
npx jscpd --config config/.jscpd.json src
```

## Tests

This config is validated indirectly by CI. The `check:dup` script runs in `test:core`.
