# .dependency-cruiser.cjs

## Purpose

[dependency-cruiser](https://github.com/sverweij/dependency-cruiser)
configuration for the AnchorStarter. The starter has a flat `src/` (no layered
sub-folders), so it enforces only the two universal structural rules rather
than the per-layer import bans used by the RecorderApp.

## Rules

- `not-to-unresolvable` (error) — fail on any import that cannot be resolved
  (catches path typos and missing files).
- `no-circular` (error) — fail on any circular dependency (the "component
  dependency loops" guard).

## Usage

```bash
pnpm run check:boundaries   # depcruise -c config/.dependency-cruiser.cjs src
```

`check:cycles` (dpdm) additionally guards cycles from the `src/main.ts` entry;
the two tools overlap on circular detection by design (defence in depth).

## Tests

No unit test; exercised by the `check:boundaries` gate inside `pnpm test`.
